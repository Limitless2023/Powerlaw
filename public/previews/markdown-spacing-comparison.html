核心发现

  • MeFlow 改前 line-height: 2.0 是所有竞品中最高的
  • 行业主流范围在 1.43 ~ 1.75 之间
  • WCAG 无障碍要求最低 1.5
  • Craft Agent 使用 1.625，与 Claude.ai 一致
  • 1.5 实测对中文偏紧凑（用户反馈拥挤）
  • 最终采用 1.625：对齐 Craft Agent 标准，中文舒适

最终方案 C 的理由 采用

维度 改前 (2.0) A: Vercel (1.43) B: Chat (1.5) C: Craft Agent (1.625) ★
中文可读性 过度留白，阅读断裂感 太紧，中文笔画密集时压迫 满足标准但中文偏紧 舒适，中文汉字有充足呼吸空间
信息密度 低（大量滚动） 很高 高（减少约 35%） 较高（减少约 25%）
WCAG 无障碍 超标 不达标（低于 1.5） 正好达标 (1.5) 达标且有余量
用户实测反馈 太松 — 中文偏紧凑（"感觉拥挤"） 与 Craft Agent 对话一致
行业对标 无对标 Vercel AI Elements — Craft Agent + Claude.ai
Tailwind 映射 leading-loose 需自定义值 leading-normal leading-relaxed（原生类名）

总结：最终采用方案 C（line-height: 1.625，p margin-bottom: 8px），对应 Tailwind 的 leading-relaxed。 该值与 Craft Agent 源码（tiptap-editor.css）和 Claude.ai 完全一致，超过 WCAG 1.5 无障碍要求，减少约 25% 的垂直空间浪费，同时保留了中文字符需要的呼吸感。 方案 B (1.5) 虽满足 WCAG 但中文实测偏紧凑（用户反馈"拥挤"），方案 A (Vercel 1.43) 对中文太紧。

迭代过程

1
初版选用方案 B (1.5) — 满足 WCAG 最低线，减少 35% 空间，理论最优
2
用户反馈 1.5 拥挤 — 实际对比 Craft Agent 对话窗口，感觉行距不够
3
调研 Craft Agent 源码 — 发现 tiptap-editor.css 使用 line-height: 1.625
4
调整为 1.625 (leading-relaxed) — 兼顾信息密度和中文阅读舒适度，对齐 Craft Agent 标准

实施方案

策略：仅在 MarkdownText.tsx 中覆盖 copilot 聊天的 markdown 样式，不修改共享的 sharedMarkdownComponents（避免影响 toast、提醒等其他消费者）。

改动范围：

  1. 1. MarkdownText.tsx — 在 defaultComponents 中覆盖 p / ul / ol / blockquote 组件，leading-relaxed (1.625)
  2. 2. Thread.tsx — 容器 leading-7 (28px) → leading-relaxed (1.625)

回退方案：所有改动集中在 copilot 组件，不影响其他 UI。如需回退只需还原 2 个文件。
