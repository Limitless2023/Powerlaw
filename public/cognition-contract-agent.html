一、认识起草智能体：你的 AI 协作伙伴

1.1 它不是"自动生成器"，而是"智能协作伙伴"

很多用户第一次接触起草智能体时，会以为它像一个表单——填好参数，点击按钮，就能拿到完美合同。

实际上，起草智能体更像一位经验丰富但刚入职的法务助手：

传统软件思维AI 协作思维
输入确定 → 输出确定描述越清晰 → 输出越精准
结果不对 = 系统 Bug结果不理想 = 可以引导调整
只能使用一次可以多轮对话持续优化
功能固定，无法变通理解上下文，灵活应对

1.2 与 AI 协作的三个核心原则

  1. 说清楚：提供充分的背景信息，AI 理解得越多，输出质量越高
  2. 分步走：不要期望一步到位，通过确认大纲 → 生成内容 → 迭代修改的节奏来协作
  3. 给反馈：对 AI 的输出给出具体反馈（"第三条违约责任部分需要加重对方的违约金比例"），而不是笼统地说"不对"

二、三大起草场景概览

MeFlow Agent 的起草智能体支持三种核心起草场景，系统会根据你的输入方式自动识别并进入对应模式：

用户发起起草请求 → 上传了合同文件？
✓ 是 → 上传文件起草
✗ 否 → 提到"模板"关键词？
✓ 是 → 智能模板推荐
✗ 否 → 从零起草（默认）
⚠️ 重要提示 只有当你明确提到"模板"、"用模板"、"找模板"、"推荐模板"等关键词时，AI 才会进入模板推荐模式。否则默认进入从零起草。

三、场景一：从零起草

3.1 适用场景

  • 没有现成模板或参考文件
  • 需要根据具体业务需求定制合同
  • 希望 AI 从需求理解开始，逐步协助完成合同起草

3.2 协作流程（7 个阶段）

1. 理解需求 → 2. 收集参考 → 3. 结构确认 → 4. 内容生成 → 5. 生成文件 → 6. 保存归档 → 7. 按需调整

3.3 各阶段详解与最佳实践

阶段 1：需求沟通（最关键的一步）

AI 会：理解合同意图、追问缺失信息、评估复杂度、判断谈判地位

✔ 好的示范 "帮我起草一份软件开发服务合同，我方是甲方（委托方）。项目是一个内部 ERP 系统的定制开发，预算 200 万，工期 6 个月，分 3 期交付验收。我们是采购方，议价能力较强。"
⚠ 一般的示范 "帮我写个合同" — AI 会追问很多问题才能开始

复杂度影响：

复杂度条款覆盖起草策略
简单核心条款快速生成
中等核心 + 风险控制分模块生成
复杂全覆盖 + 专项条款并行模块化生成

阶段 2：参考资料收集

AI 自动搜索历史合同、模板库、范本库、条款库，为起草提供参考依据。

阶段 3：大纲确认（重要检查点）

AI 给出合同结构大纲，等待你的确认。

💡 提示 在大纲阶段调整成本极低。务必仔细检查结构是否符合需求，此时修改远比生成全文后再改高效。

阶段 4：内容生成

根据确认的大纲生成合同全文，使用专业法律语言。

阶段 5：文件交付

转换为 Word 文档交付审阅。

阶段 6：系统保存与表单填写

创建合同记录、上传文件、协助填写表单。

阶段 7：迭代修改

接受修改请求，精准修改而非整篇重写。

✔ 好的反馈 "第五条违约责任中，逾期付款的违约金比例从每日万分之三改为每日万分之五，并增加一条：连续逾期超过30天，我方有权解除合同。"
⚠ 低效的反馈 "违约部分不太好，改一下"

四、场景二：智能模板推荐

4.1 适用场景

  • 公司已有标准合同模板
  • 希望快速找到合适的模板
  • 适合标准化程度高的合同

4.2 如何触发

需明确提到模板相关关键词：

✔ 触发模板模式 "帮我找一个采购合同模板"
✗ 不会触发（进入从零起草） "帮我起草一份采购合同"

4.3 协作流程

1. 明确意图 → 2. 匹配模板 → 3. 关联模板 → 4. 填写信息

五、场景三：上传文件起草

5.1 适用场景

  • 已有现成合同文件
  • 需要导入系统并录入信息
  • AI 不会修改上传的文件内容

5.2 协作流程

1. 文件确认 → 2. 上传关联 → 3. 是否填表 → 4. 提取填写

六、与 AI 高效协作的实用技巧

6.1 提高输出质量的沟通方式

  • 明确立场：告知你方角色和谈判地位
  • 提供背景：说明业务场景和特殊要求
  • 指定参考：提供参考合同或条款
  • 具体反馈：针对具体条款给出修改意见
  • 说明优先级：表达哪些条款最重要

6.2 常见误区

误区正确认知
"AI 应该一次就写对"把第一次输出当起点，2-3 轮迭代是正常节奏
"说得越简单越好"首次对话尽可能多提供信息
"输出不对就是系统 Bug"先尝试更具体的反馈引导 AI 调整
"AI 替我做所有决定"认真审阅每个确认环节，专业判断仍然是你的

6.3 什么情况下联系技术支持

🚨 真正的技术问题

  • 系统报错 / 页面崩溃
  • 格式错乱
  • AI 无响应
  • 功能缺失
  • 数据保存失败

🔄 先调整协作方式

  • 条款不够专业
  • 缺少某些条款
  • 语气/立场不对
  • 内容与预期不符

七、快速参考卡片

三种模式的快速启动模板：

📄 从零起草

适合定制化需求，AI 从理解意图开始全程协作。

快速启动：
"帮我起草一份 [合同类型]，我方是 [甲方/乙方]。[简述业务背景]，预算 [金额]，工期 [时长]。"

📑 模板起草

适合标准化合同，快速匹配已有模板。

快速启动：
"帮我找一个 [合同类型] 的模板，用于 [业务场景]。"

📤 上传文件起草

适合已有文件的系统录入。

快速启动：
直接上传合同文件，AI 会自动识别并引导后续流程。
